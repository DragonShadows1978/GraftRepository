## Seam (already characterized — E2E receipts 2026-07-08)

Supersession fails at ROUTING, not readback: under competition the
correction node loses the route to a value-bearing competitor
(Orion probe: wrong node scored 1.000, pristine correct memory ranked
4th; readout exonerated). Decomposed cause: PRODUCT SCORING with
MAX-POOL LENGTH BIAS — long turn keys dominate short fact keys (a
zero-overlap filler outranked the 15-token fact memory; Δ 100%
latent). The stale value itself NEVER resurfaced (no un-supersession
at fp16 — that failure mode belongs to sub-floor storage quant).
Registered successor options from the E2E queue: length-debias /
kind-prior; supersession probe battery.

## Two registered levers

- **L1 route length-debias:** normalize the latent route score's
  length sensitivity (candidate forms: length-normalized max-pool;
  mean-over-top-m pairs; kind-prior is EXPLICITLY OUT — it is an
  importance-flavored route bias and the attractor law stands).
  Chosen form must be justified against the E2E decomposition and
  frozen before G1 runs.
- **L2 revision-aware mount resolution:** supersedes-lineage is
  CORRECTNESS metadata, not importance. At mount-set assembly (after
  routing, before injection): if node A supersedes node B (explicit
  supersede metadata per the M5 machinery, or same-node_id lineage)
  and A is available, B is excluded from the mount set (B remains
  routable evidence that the topic exists; descent may still reach
  it explicitly). Readout never sees a superseded node co-mounted
  with its successor.

## Gates

- **G0-SUP (battery + local baseline):** dedicated probe battery
  (fixtures): scripted sessions with fact→correction→competition
  scenarios — short corrections vs long value-bearing competitors,
  multi-hop supersession (A→B→C), correction-then-restatement, and
  fresh-fact controls. Run UNPATCHED on MiniCPM3: baseline route
  ranks + answer correctness. This measures whether the GQA-observed
  seam transfers to MLA and sets the floors. Numeric thresholds for
  G1/G2 registered after G0-SUP, before either runs.
- **G1-SUP (L1 length-debias):** battery re-run with debias only.
  Demand: correction routes ≥ its pre-fix rank in every competition
  scenario, competitor-over-correction inversions eliminated or
  reported case-by-case; ALL existing routing regressions green
  (arena 6/6, trips 6/6, corpus 20/20 — the debias touches every
  route, so the whole recall surface re-gates).
- **G2-SUP (L2 resolution):** battery with L1+L2. Demand: current
  value returned, stale never returned, across the battery;
  multi-hop lineage resolves to the head; descent/readback of
  superseded nodes still possible when explicitly requested.
- **G3-SUP (composed):** predecessor E2E probes (Orion-class
  scenario) reproduced on this stack and GREEN.

## Non-goals

- No kind-priors or importance terms in routing (attractor law).
- No un-supersession work (storage-quant law owns that; floors hold).
- GPT-OSS/GQA port of the fix = successor, not scope.
